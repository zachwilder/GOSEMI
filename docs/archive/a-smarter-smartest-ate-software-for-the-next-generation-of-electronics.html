Top Stories • 2017

With smaller transistors that pack more and more functionality on a single chip, the complexity of test programs is increasing apace with that of the ICs themselves.

Operating Sequence

Advanced multi-domain devices consist of multiple different functional blocks. Typical block types include an RF block for transmitting/receiving phone signals, a protocol-ware (PA) interface to condition the device, analog blocks for microphone use and playing music, and/or digital blocks for processing.

Testing one functional block, e.g. the RF block, typically requires â€˜assembling a testâ€™ out of multiple pieces, e.g., starting an analog stimulus signal, conditioning the device with digital signals, and starting one or multiple RF measurements. The Operating Sequence is designed to easily assemble the multiple â€˜test pieces,â€™ with precise synchronization where needed. These test pieces are typically patterns, protocol transactions, or â€˜actions.â€™ Actions can be DC stimulus changes (e.g., stepping up a ramp), DC measurements, analog stimulus and measurements, or RF stimulus and measurements â€“ to name some examples.
